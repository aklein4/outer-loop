# Conditional Latent Variable Notes

Dataset: `aklein4/few-shot-TrackStar-8192-test`
Config: `HuggingFaceTB--smoltalk--train`
Split: `train`
Sampling: 10 clusters sampled from cluster IDs `0..99` with `random.Random(42)`.
Generation script: `data_preparation/few_shot/view_clusters.py`

The sampled clusters are `3`, `13`, `14`, `17`, `28`, `31`, `35`, `81`, `86`, and `94`. Each saved JSON contains 64 examples and has only `system`, `user`, and `response` fields. `system` is `null` when no system prompt is present.

This pass looks for latent variables that govern `p(y | x)`: response behavior, transformation rules, output policy, reasoning trace style, tone, format, or other information that changes the distribution over valid responses after the input is known. Pure facts about `p(x)`, `p(y)`, or broad `p(x, y)` subject matter are not counted as defining variables unless they also predict how the answer is written conditional on the prompt.

Theme-fit counts are judgments against the conditional latent variable, counting examples where seeing other examples from the cluster would plausibly help predict `y` given `x`.

## Meta-Learning Assessment

Cluster `028` is the best aligned with the meta-learning objective in this sample. Its latent variable directly constrains `p(y | x)`: for a known elementary counting or divisibility prompt, the cluster strongly predicts an enumerative solution style with explicit candidate checking and a concise final count.

Clusters `003`, `013`, and `014` are also useful. They mostly reveal response policy rather than just topic: worked algebra derivation, programming implementation/debugging help, and data-analysis consulting. A support set from these clusters should reduce uncertainty about how detailed, procedural, and scaffolded the response should be.

Clusters `017`, `031`, `035`, and `094` carry moderate but less clean conditional signal. They identify broad response modes such as ideation, logistics planning, or speculative exposition, but those modes are entangled with prompt topic and mixed with unrelated examples.

Clusters `081` and `086` are weak for meta-learning in the 8192 sample. Each contains pockets with real conditional behavior, but no single hidden variable reliably governs the full cluster. This is a notable reversal from the smaller-sample analysis, where `086` was a clean tone-transfer cluster and `081` was a clean code-block implementation cluster.
